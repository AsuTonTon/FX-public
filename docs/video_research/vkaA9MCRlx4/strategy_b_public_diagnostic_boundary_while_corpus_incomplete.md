# Strategy B Public Diagnostic Boundary While Corpus Incomplete

## 1. Objective

This public-repository boundary note pauses further Strategy B local-only
diagnostic authorization, readiness, packet, sample, Pass A / Pass B, and
review-run automation while channel-wide corpus input, visual evidence review,
and doctrine synthesis remain incomplete.

The goal is to prevent public Strategy B diagnostic-readiness momentum from
turning incomplete-input observations into local diagnostic execution, Strategy
B rule changes, parameter changes, Strategy B2 or Strategy C proposals, or
implementation work.

This note uses committed `AsuTonTon/FX-public` repository docs only. It does
not rely on private repository content or external sources.

## 2. Current Public-Repo Diagnostic-Readiness State

The public repo contains a Strategy B diagnostic-readiness stack under
`docs/video_research/vkaA9MCRlx4/`.

Current public state:

- The diagnostic authorization template defines what a future bounded
  local-only diagnostic issue would need before any local read or diagnostic
  run.
- The low-R structural TP issue skeleton and review checklist define how a
  future issue could be reviewed for completeness.
- The design-review gate, evidence-gap, stop-width versus stop-validity,
  contextual-RR, and taxonomy docs preserve a Strategy B behavior freeze.
- The remaining-open-items triage keeps low-R structural TP, stop-anchor
  quality, stop-width versus stop-validity, no-valid-target, and
  no-trade-structure questions separated.
- The current public source set does not include a separate tracked low-R
  structural TP packet readiness note. The review checklist recommends that as
  a possible future docs-only task, not as an existing readiness approval.
- Public guardrails in `AGENTS.md` and `.github/workflows/fx-guardrails.yml`
  keep raw data, generated artifacts, local paths, source bodies, external
  access, and high-risk claims out of the public repository.

This readiness stack is useful for planning. It does not authorize local-only
execution while the broader corpus, visual evidence, and doctrine-synthesis
inputs remain incomplete.

## 3. Why Public Diagnostic Momentum Is Risky

Public diagnostic momentum is risky because readiness documents can make a
future local-only diagnostic look routine before the evidence base is complete.

Specific risks:

- a skeleton or checklist can be mistaken for approval to run a diagnostic;
- packet or sample planning can drift into inspecting local artifacts;
- Pass A / Pass B mechanics can imply that a review run should begin;
- low-R structural TP labels can collapse no-valid-target, no-trade-structure,
  stop-width, and stop-validity categories;
- aggregate observations can create pressure to change Strategy B rules or
  parameters;
- incomplete channel corpus themes can be imported directly into executable
  Strategy B behavior;
- Strategy B2 or Strategy C can be proposed as a shortcut around unresolved
  evidence gaps.

The safe interpretation is narrower: public docs may preserve boundaries,
name blockers, and define what future approval would require. They may not
start diagnostic execution or implementation work.

## 9. Strategy B2 / Strategy C Prohibition

This boundary prohibits using incomplete public diagnostic-readiness work to
create or advance Strategy B2 or Strategy C.

Do not:

- name a new strategy variant from this diagnostic line;
- draft Strategy B2 or Strategy C behavior;
- route low-R structural TP, stop-width, stop-validity, no-valid-target, or
  no-trade-structure questions into a new strategy design;

- use a local-only diagnostic blocker as evidence for a replacement strategy.

executive decision opens that question.
